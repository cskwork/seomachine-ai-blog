## 코드에서 캔버스로, 역방향 파이프라인의 등장

디자인에서 코드로 넘어가는 흐름은 익숙하다. 그 반대 방향은 어떨까.

Figma가 2026년 2월 공개한 **Code to Canvas** 기능은 Claude Code로 빌드한 실제 작동 UI를 Figma 캔버스에 편집 가능한 프레임으로 변환합니다. 스크린샷이나 평면 이미지가 아니라, 레이어 구조와 Auto Layout이 살아 있는 진짜 디자인 아티팩트다.

> 이 글은 Figma 공식 블로그 ["From Claude Code to Figma: Turning Production Code into Editable Figma Designs"](https://www.figma.com/blog/introducing-claude-code-to-figma/)를 바탕으로 한국어로 재구성한 가이드입니다.

### 왜 지금 이 기능이 필요한가

AI 코딩 도구가 보편화되면서 병목이 이동했습니다. "이걸 어떻게 만들지?"에서 **"어떤 버전을 출시할지?"**로 질문이 바뀐 것입니다. Claude Code, Cursor, Windsurf 같은 도구로 수분 만에 작동하는 인터페이스를 만들 수 있지만, 여러 대안을 나란히 놓고 비교하고 결정하는 과정은 여전히 Figma 캔버스에서 이루어집니다.

Code to Canvas는 이 빠진 고리를 채운다. 코드로 빌드한 결과물을 팀이 함께 보고 판단하는 공유 공간으로 되돌려 보냅니다.

---

## Code to Canvas 작동 원리

핵심 워크플로우는 네 단계로 구성됩니다.

### 1단계: 빌드 또는 반복

Claude Code로 UI를 만들거나 수정합니다. 로컬 개발 서버, 스테이징, 프로덕션 등 브라우저에서 실행되는 모든 인터페이스가 대상입니다.

### 2단계: 화면 캡처

통합 기능이 실시간 브라우저 상태를 캡처하여 Figma 호환 프레임 형식으로 변환합니다. Claude Code에서 "Send this to Figma" 명령을 실행하면 됩니다.

### 3단계: Figma에 붙여넣기

캡처된 화면이 캔버스에 편집 가능한 디자인 아티팩트로 배치됩니다. 평면 이미지가 아닌, 구조가 살아 있는 프레임입니다.

### 4단계: 협업

팀이 캔버스에서 주석을 달고, 복제하고, 재배치하고, 대안을 비교합니다. 코드 접근 권한 없이도 모든 작업이 가능하다.

### 멀티 스크린 세션

온보딩, 결제, 설정 등 전체 플로우를 한 세션에서 캡처하여 캔버스에 배치할 수 있습니다. 팀은 프레임을 복제해 구조 변경을 실험하고, 여러 대안을 비교합니다. 기각된 아이디어도 나중에 참조할 수 있도록 캔버스에 남겨둘 수 있습니다.

## Figma MCP 서버 설정 방법

Code to Canvas는 Figma의 **MCP(Model Context Protocol) 서버** 위에서 동작합니다. MCP는 AI 도구가 외부 데이터 소스 및 애플리케이션과 연결하는 개방형 표준입니다.

### 설정 과정 (3단계)

**1단계: MCP 서버 활성화**

Figma 데스크톱 앱의 환경설정에서 "Dev Mode MCP Server"를 활성화합니다. 로컬에서 `http://127.0.0.1:3845/sse` 주소로 실행됩니다.

**2단계: Claude Code 연결**

터미널에서 다음 명령어를 실행한다:

```bash
claude mcp add --transport sse figma-dev-mode-mcp-server http://127.0.0.1:3845/sse
```

**3단계: 작업 시작**

Figma 데스크톱 앱에서 프레임을 선택하거나, 디자인 링크를 Claude Code 프롬프트에 붙여넣어 작업을 시작합니다.

### 요구 사항

- Figma 데스크톱 앱 (브라우저 버전 미지원)
- Figma Dev 또는 Full 라이선스
- Claude Code (npm으로 설치)

## 코드 없이 디자이너가 할 수 있는 것

코드로 만든 UI가 Figma 캔버스에 도착하면, 디자이너는 익숙한 환경에서 바로 작업할 수 있습니다.

### 나란히 비교

AI가 생성한 여러 변형을 캔버스에 나란히 배치합니다. 플로우 전반의 패턴, 공백, 불일치를 한눈에 파악할 수 있습니다.

### 구조 탐색

프레임을 복제하고 단계를 재배치하며 레이아웃 변경을 실험합니다. 정보 계층 구조의 대안을 탐색할 때 코드 지식은 필요 없습니다.

### 주석과 피드백

근사치가 아닌, 실제로 빌드된 인터페이스 위에 코멘트를 남긴다. PM, 디자이너, 엔지니어가 동일한 산출물을 보며 소통합니다.

### 디자인 시스템 정합성 검증

AI가 생성한 UI가 기존 컴포넌트, 토큰, 패턴과 일치하는지 확인합니다. 프로덕션에 반영되기 전에 불일치를 잡아낼 수 있습니다.

---

## 양방향 워크플로우: 캔버스에서 코드로 돌아가기

반대 방향도 동일하게 중요하다. Figma에서 프레임을 선택하고 Claude Code에 링크를 전달하면, Claude가 디자인 시스템을 반영한 프로덕션 코드를 생성합니다. 컴포넌트, 토큰, Tailwind 변수를 읽어서 시스템에 맞는 코드를 생성합니다.

### 라운드 트립 순환

```
Figma에서 디자인
  → Claude Code로 코드 생성
    → 작동하는 UI를 다시 Figma로 캡처
      → 캔버스에서 다듬기
        → 수정사항을 코드로 반영
```

매 사이클에서 컨텍스트가 보존됩니다. 동일한 시스템 오브 레코드(MCP)가 양쪽 환경을 연결하기 때문입니다. 디자인과 개발 사이 닫힌 루프에 가장 가까운 워크플로우다.

---

## Code to Canvas vs Figma Make vs Figma MCP 비교

| 항목 | Code to Canvas | Figma Make | Figma MCP 서버 |
|------|---------------|------------|---------------|
| **방향** | 코드 → 디자인 | 텍스트/디자인 → 코드 | 양방향 컨텍스트 레이어 |
| **시작점** | 브라우저에서 작동 중인 UI | 프롬프트 또는 기존 디자인 | Figma 프레임 또는 Claude Code 세션 |
| **출력** | 편집 가능한 Figma 프레임 | 프론트엔드 코드 또는 프로토타입 | AI 도구용 구조화된 디자인 컨텍스트 |
| **주 사용자** | 개발자, 테크니컬 디자이너 | 디자이너, 비개발 직군 | 양쪽 모두 (인프라 역할) |
| **최적 용도** | AI로 빌드한 프로토타입을 캔버스에서 팀 리뷰 | 디자인이나 자연어에서 코드 생성 | 디자인 시스템을 AI 코딩 도구에 연결 |

세 기능은 경쟁 관계가 아니라 상호 보완적입니다. Figma Make는 비개발자에게 접근성이 높고, Code to Canvas는 이미 터미널에서 웹 애플리케이션을 빌드하는 팀에 최적화되어 있습니다.

## 알려진 한계점과 대처 방법

강력하지만 모든 상황에 적합하지는 않다.

### 터미널 중심 워크플로우

Claude Code는 명령줄에서 실행됩니다. 터미널에 익숙하지 않은 디자이너는 초기 설정 단계에서 지원이 필요하다.

### 시각적 미세 조정의 부재

코드로 돌아간 후에는 패딩이나 호버 상태를 조정하려면 직접 코드를 편집해야 합니다. 도구 내 시각적 피드백 루프는 아직 지원되지 않습니다.

### 멀티 프레임 복잡성

여러 화면으로 구성된 플로우를 변환하려면 각 화면을 개별로 캡처한 뒤 조합해야 합니다. 복잡한 프로젝트에서는 시간과 토큰 비용이 늘어난다.

### 코드베이스 직접 수정

Claude Code는 실제 코드베이스에서 작동합니다. 변경사항이 엔지니어가 배포하는 파일에 그대로 반영되므로 주의가 필요하다.

### 데스크톱 앱 필수

MCP 서버는 Figma 데스크톱 앱에서만 실행됩니다. 브라우저 버전에서는 사용할 수 없습니다.

### 토큰 비용

디자인 파일이 크거나 멀티 스크린 플로우일수록 토큰 소비가 증가합니다. 복잡한 프로젝트에서는 비용을 미리 계획해야 합니다.

## 디자인-개발 핸드오프의 미래

이 기능의 의미는 하나의 기능을 넘어선다. 디자인 도구와 코딩 도구가 경쟁이 아닌 통합 시스템의 구성 요소로 수렴하는 흐름입니다.

Figma의 관점에서 AI는 캔버스를 대체하는 것이 아니라, 더 많은 선택지를 더 빠르게 캔버스에 공급하는 역할입니다. 디자이너의 역할은 산출물 제작에서 AI가 생성한 결과를 선별하고 정제하는 방향으로 이동합니다.

**코드는 수렴에 강하다.** 빌드를 실행하고, 경로를 따라가고, 한 번에 하나의 상태에 도달합니다.
**캔버스는 발산에 강하다.** 전체 경험을 배치하고, 분기를 보고, 방향을 함께 정합니다.

Code to Canvas는 이 두 영역을 연결합니다.
