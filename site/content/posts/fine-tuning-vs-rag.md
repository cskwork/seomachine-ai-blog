## 문제 정의

범용 LLM은 일반적인 지식에는 강하지만, 특정 도메인이나 조직의 데이터에 대해서는 한계가 있습니다.

이 문제를 해결하는 두 가지 주요 접근법이 있습니다:

- **파인튜닝(Fine-Tuning)**: 모델 자체를 도메인 데이터로 추가 학습
- **RAG(Retrieval-Augmented Generation)**: 추론 시점에 관련 데이터를 검색하여 제공

## 원리 비교

### 파인튜닝

기존 모델의 가중치를 도메인 데이터셋으로 업데이트합니다.

```
기존 모델 + 도메인 학습 데이터 → 학습 → 커스터마이징된 모델
```

모델이 도메인 지식을 내재화하여, 추론 시 별도의 검색 없이 해당 지식을 활용합니다.

### RAG

모델은 변경하지 않고, 추론 시점에 외부 지식을 검색하여 프롬프트에 포함합니다.

```
사용자 질의 → 관련 문서 검색 → [질의 + 검색 결과]를 모델에 전달 → 응답
```

모델은 제공된 컨텍스트를 기반으로 답변을 생성합니다.

## 비교 분석

| 기준 | 파인튜닝 | RAG |
|------|---------|-----|
| 초기 비용 | 높음 (GPU, 학습 시간) | 낮음 (벡터 DB 설정) |
| 운영 비용 | 모델 호스팅 비용 | 검색 + API 호출 비용 |
| 데이터 업데이트 | 재학습 필요 | 문서 업데이트로 즉시 반영 |
| 환각 제어 | 제한적 | 출처 기반 응답으로 제어 용이 |
| 출처 추적 | 불가능 | 가능 (검색된 문서 참조) |
| 도메인 톤/스타일 | 우수 | 프롬프트 의존 |
| 구현 복잡도 | 높음 | 중간 |
| 데이터 요구량 | 수천-수만 건 | 제한 없음 |

## 사용 사례별 추천

### 파인튜닝이 적합한 경우

**1. 특수한 출력 형식이 필요한 경우**

- 의료 보고서 작성
- 법률 문서 초안
- 특정 코딩 스타일 준수

학습 데이터의 패턴을 모델이 내재화하므로, 일관된 형식의 출력을 생성합니다.

**2. 도메인 전문 용어와 맥락 이해**

- 산업별 전문 용어
- 조직 내부 약어와 개념
- 특정 기술 스택의 관용구

**3. 응답 톤과 스타일 커스터마이징**

- 브랜드 보이스 반영
- 특정 대상 독자에 맞는 설명 수준
- 조직 고유의 커뮤니케이션 스타일

### RAG가 적합한 경우

**1. 최신 정보가 중요한 경우**

- 제품 문서, FAQ
- 뉴스, 시장 동향
- 정책, 규정 변경

파인튜닝된 모델의 지식은 학습 시점에 고정됩니다. RAG는 문서를 업데이트하면 즉시 최신 정보를 반영합니다.

**2. 출처 추적이 필요한 경우**

- 규정 준수 (응답의 근거 제시)
- 고객 지원 (참조 문서 안내)
- 리서치 (인용 출처 명시)

**3. 대규모 지식베이스 활용**

- 수천 페이지의 기술 문서
- 과거 이슈 및 해결 사례 DB
- 다양한 제품의 사양 정보

## 하이브리드 접근법

실무에서는 파인튜닝과 RAG를 결합하는 것이 가장 효과적인 경우가 많습니다.

```
파인튜닝: 도메인 톤/스타일 + 전문 용어 이해
    +
RAG: 최신 데이터 + 구체적 사실 정보 + 출처 추적
    =
도메인 전문가 수준의 최신 정보 기반 응답
```

### 하이브리드 적용 예시

의료 AI 어시스턴트:
- 파인튜닝: 의료 용어, 진단 보고서 형식, 의사와 환자 간 소통 톤
- RAG: 최신 의학 논문, 약물 정보, 진료 가이드라인

코드 어시스턴트:
- 파인튜닝: 팀의 코딩 컨벤션, 아키텍처 패턴, 코드 리뷰 스타일
- RAG: 프로젝트 코드베이스, 이슈 히스토리, 기술 문서

## 의사결정 플로우차트

```
최신 정보가 중요한가?
├── 예 → RAG (또는 하이브리드)
└── 아니오
    ↓
출처 추적이 필요한가?
├── 예 → RAG (또는 하이브리드)
└── 아니오
    ↓
특수한 출력 형식/스타일이 필요한가?
├── 예 → 파인튜닝 (또는 하이브리드)
└── 아니오
    ↓
프롬프트 엔지니어링만으로 충분한가?
├── 예 → 프롬프트 엔지니어링
└── 아니오 → 하이브리드
```

## 실무 조언

1. **RAG부터 시작하세요**: 구현이 빠르고, 데이터 업데이트가 용이합니다
2. **RAG로 부족한 부분을 파악하세요**: 톤, 형식, 전문 용어에서 한계를 느끼면 파인튜닝 검토
3. **파인튜닝 데이터를 꾸준히 수집하세요**: 사용자 피드백, 수정된 응답을 학습 데이터로 축적
4. **비용을 지속 모니터링하세요**: 두 접근법의 TCO(총 소유 비용)를 비교
