포트폴리오 사이트를 만들었습니다. HTML, CSS까지 어렵게 완성했습니다. 그런데 이걸 인터넷에 올리는 방법을 모릅니다. 서버? 호스팅? FTP? 들어본 적은 있지만 어디서부터 시작해야 할지 막막합니다.

사실 2026년 현재, 웹사이트를 인터넷에 올리는 데 서버 지식은 필요 없습니다. **Vercel**이라는 플랫폼을 쓰면, GitHub에 파일을 올리고 버튼 몇 번 클릭하는 것만으로 전 세계 어디서나 접속 가능한 웹사이트가 만들어집니다. 무료입니다.

이 글은 코딩 경험이 전혀 없는 마케터, 디자이너, 학생, 1인 창업자를 위해 작성했습니다. 터미널 명령어를 외울 필요 없습니다. 각 단계를 순서대로 따라하면 됩니다.

## 먼저 알아야 할 것: 정적 사이트란 무엇인가

웹사이트는 크게 두 종류입니다.

**동적 사이트**는 서버가 매번 페이지를 만들어 보내줍니다. 네이버, 쿠팡 같은 사이트가 여기에 해당합니다. 로그인 상태에 따라 다른 화면을 보여줘야 하니까요.

**정적 사이트**는 미리 만들어진 HTML 파일을 그대로 보여줍니다. 포트폴리오, 회사 소개 페이지, 블로그, 이벤트 랜딩 페이지가 대표적입니다. 서버에서 별도 처리를 할 필요가 없으니 빠르고, 안전하고, 유지비가 거의 들지 않습니다.

정리하면 이렇습니다.

| 구분 | 동적 사이트 | 정적 사이트 |
|------|------------|------------|
| 예시 | 쇼핑몰, SNS, 대시보드 | 포트폴리오, 블로그, 랜딩 페이지 |
| 서버 필요 | 반드시 필요 | 불필요 |
| 속도 | 서버 처리 시간만큼 느림 | 매우 빠름 |
| 비용 | 월 수만 원–수십만 원 | 대부분 무료 |
| 보안 | 서버 관리 필요 | 공격 표면 거의 없음 |

내 사이트가 정적 사이트인지 확인하는 방법은 간단합니다. 폴더 안에 `index.html` 파일이 있고, 그 파일을 브라우저로 열면 사이트가 보인다면 정적 사이트입니다.

## Vercel이 비개발자에게 최적인 3가지 이유

정적 사이트를 올릴 수 있는 플랫폼은 여러 개 있습니다. Netlify, GitHub Pages, Cloudflare Pages, AWS S3 등. 그런데 비개발자에게 Vercel을 추천하는 이유는 명확합니다.

### 1. 설정이 거의 없습니다

GitHub 저장소를 연결하면 Vercel이 자동으로 프로젝트 유형을 감지합니다. 빌드 명령어, 출력 폴더 같은 설정을 알아서 채워줍니다. 비개발자가 가장 막히는 지점이 "설정"인데, Vercel은 이 단계를 사실상 없앴습니다.

### 2. 무료 범위가 넉넉합니다

Hobby 플랜(무료)으로 월 100GB 대역폭, 무제한 배포, HTTPS 자동 적용, 커스텀 도메인 연결까지 가능합니다. 개인 프로젝트나 소규모 비즈니스 사이트라면 유료 플랜이 필요 없습니다.

### 3. 수정하면 자동으로 반영됩니다

GitHub에 파일을 수정해서 올리면, Vercel이 알아서 새 버전을 배포합니다. FTP로 파일을 일일이 올리던 시대는 끝났습니다. 이것을 **지속적 배포(Continuous Deployment)**라고 부르는데, 원리를 몰라도 됩니다. 그냥 파일 수정하고 저장하면 사이트에 반영된다는 것만 기억하면 됩니다.

## 사전 준비: GitHub 계정 만들기

Vercel 배포에는 GitHub 계정이 필요합니다. 이미 있다면 이 섹션을 건너뛰어도 됩니다.

### GitHub 가입 절차

1. [github.com](https://github.com)에 접속합니다.
2. 우측 상단 **Sign up** 버튼을 클릭합니다.
3. 이메일, 비밀번호, 사용자명을 입력합니다. 사용자명은 영문과 숫자 조합으로 짧게 만드는 것이 좋습니다.
4. 이메일 인증을 완료합니다.
5. 무료 플랜(Free)을 선택합니다.

GitHub는 코드 저장소 서비스입니다. 파일을 올려두면 버전 관리가 되고, Vercel 같은 배포 서비스와 연동할 수 있습니다. 비개발자 입장에서는 "내 웹사이트 파일을 보관하는 구글 드라이브"라고 이해하면 됩니다.

### 웹사이트 파일을 GitHub에 올리기

터미널 없이 GitHub 웹사이트에서 직접 올리는 방법입니다.

1. GitHub에 로그인한 뒤, 우측 상단 **+** 버튼 → **New repository**를 클릭합니다.
2. Repository name에 프로젝트 이름을 입력합니다. 예: `my-portfolio`
3. **Public**을 선택합니다. (무료 플랜에서 Vercel 배포에 필요합니다.)
4. **Add a README file**에 체크합니다.
5. **Create repository** 버튼을 클릭합니다.
6. 생성된 저장소 페이지에서 **Add file** → **Upload files**를 클릭합니다.
7. 내 컴퓨터에 있는 웹사이트 폴더의 파일들을 드래그 앤 드롭합니다. (`index.html`, CSS 파일, 이미지 등)
8. 하단의 **Commit changes** 버튼을 클릭합니다.

**주의사항**: 폴더째로 올리면 폴더 구조가 유지됩니다. `index.html`이 저장소의 최상위(루트)에 있어야 합니다. 만약 `my-site/index.html`처럼 하위 폴더에 들어가 있다면, Vercel에서 별도 설정이 필요합니다. 가장 간단한 방법은 폴더 안의 파일들만 직접 올리는 것입니다.

## 핵심: Vercel로 배포하기 (5단계)

GitHub에 파일을 올렸다면, 이제 Vercel에서 배포할 차례입니다. 총 5단계이며, 각 단계는 1–2분이면 끝납니다.

### 1단계: Vercel 가입

1. [vercel.com](https://vercel.com)에 접속합니다.
2. 우측 상단 **Sign Up** 버튼을 클릭합니다.
3. **Continue with GitHub**를 선택합니다. 별도 계정을 만들 필요 없이 GitHub 계정으로 바로 가입됩니다.
4. GitHub 연동 권한을 승인합니다.

이메일로 가입할 수도 있지만, GitHub 연동이 필수이므로 처음부터 GitHub으로 가입하는 것이 한 단계를 줄입니다.

### 2단계: 새 프로젝트 만들기

1. Vercel 대시보드에서 **Add New...** → **Project**를 클릭합니다.
2. **Import Git Repository** 화면이 나타납니다.
3. 방금 만든 GitHub 저장소(예: `my-portfolio`)가 목록에 보입니다. **Import** 버튼을 클릭합니다.

저장소가 보이지 않는다면, **Adjust GitHub App Permissions**를 클릭해서 해당 저장소에 대한 접근 권한을 추가합니다. 이것은 보안을 위해 Vercel이 모든 저장소가 아닌 선택한 저장소만 접근하도록 설계된 것입니다.

### 3단계: 프로젝트 설정

Import 후 설정 화면이 나타납니다. 정적 사이트라면 대부분의 설정을 건드릴 필요가 없습니다.

**확인할 항목:**

- **Framework Preset**: `Other`로 되어 있으면 정상입니다. HTML/CSS만 있는 정적 사이트이기 때문입니다.
- **Build Command**: 비워둡니다. 빌드 과정이 필요 없습니다.
- **Output Directory**: 비워두거나 `.`(점)으로 설정합니다. 저장소 루트가 곧 사이트입니다.
- **Install Command**: 비워둡니다. 설치할 패키지가 없습니다.

Hugo, Jekyll, Astro 같은 정적 사이트 생성기(SSG)를 사용하는 경우에는 Vercel이 자동으로 프레임워크를 감지합니다. 이 경우 설정이 자동으로 채워지니 그대로 두면 됩니다.

### 4단계: 배포

**Deploy** 버튼을 클릭합니다. 끝입니다.

Vercel이 GitHub에서 파일을 가져와 배포를 시작합니다. 정적 사이트는 보통 10–30초면 배포가 완료됩니다. 화면에 "Congratulations!" 메시지와 함께 사이트 미리보기가 나타납니다.

### 5단계: 사이트 확인

배포가 완료되면 Vercel이 자동으로 URL을 생성합니다. 형식은 `프로젝트명.vercel.app`입니다. 예를 들어 `my-portfolio.vercel.app`으로 바로 접속할 수 있습니다.

이 URL은 영구적입니다. HTTPS가 자동 적용되어 있어 보안 경고 없이 안전하게 접속됩니다.

## 사이트 수정하는 법: 파일 바꾸면 자동 반영

사이트를 수정하고 싶을 때, Vercel 대시보드에서 뭔가를 할 필요가 없습니다. GitHub에서 파일을 수정하면 Vercel이 자동으로 새 버전을 배포합니다.

### GitHub 웹에서 직접 수정하기

1. GitHub의 저장소 페이지로 이동합니다.
2. 수정하려는 파일(예: `index.html`)을 클릭합니다.
3. 우측 상단의 연필 아이콘(Edit this file)을 클릭합니다.
4. 내용을 수정합니다.
5. **Commit changes** 버튼을 클릭합니다.

커밋(Commit)이 완료되면 Vercel이 즉시 변경을 감지하고 새 버전을 배포합니다. 보통 30초–1분 이내에 사이트에 반영됩니다.

### 파일을 교체하는 경우

이미지를 교체하거나 새 파일을 추가하려면, 저장소 페이지에서 **Add file** → **Upload files**를 사용합니다. 같은 이름의 파일을 올리면 자동으로 덮어씌워집니다.

## 커스텀 도메인 연결하기

`my-portfolio.vercel.app`도 좋지만, 본인만의 도메인(`myname.com`)을 사용하고 싶을 수 있습니다. Vercel에서 커스텀 도메인을 연결하는 과정은 기술적이지만, 천천히 따라하면 충분히 할 수 있습니다.

### 도메인이 없는 경우: 도메인 구매

도메인은 연간 1–2만 원 정도입니다. 국내에서는 가비아, 카페24가 대표적이고, 해외에서는 Namecheap, Cloudflare Registrar가 많이 사용됩니다. Vercel 자체에서도 도메인을 구매할 수 있습니다.

구매 시 `.com`, `.kr`, `.io` 등 다양한 확장자를 선택할 수 있습니다. 개인 포트폴리오라면 `.com`이나 `.dev`를, 사업용이라면 `.com`이나 `.co.kr`을 추천합니다.

### Vercel에 도메인 등록

1. Vercel 대시보드에서 프로젝트를 클릭합니다.
2. **Settings** → **Domains**로 이동합니다.
3. 구매한 도메인 주소(예: `myname.com`)를 입력하고 **Add**를 클릭합니다.
4. Vercel이 DNS 설정 안내를 보여줍니다.

### DNS 설정하기

DNS 설정이 이 과정에서 가장 어려운 부분입니다. 하지만 Vercel이 정확히 어떤 값을 어디에 입력해야 하는지 알려주므로, 그대로 따라하면 됩니다.

**Vercel이 안내하는 두 가지 방식 중 하나를 선택합니다:**

**방식 A: A 레코드 (추천)**

- 도메인 구매처(가비아, Namecheap 등)의 DNS 관리 페이지에 접속합니다.
- **A 레코드**를 추가합니다.
- Host(호스트): `@`
- Value(값): Vercel이 안내하는 IP 주소 (예: `76.76.21.21`)

**방식 B: CNAME 레코드**

- **CNAME 레코드**를 추가합니다.
- Host(호스트): `www`
- Value(값): `cname.vercel-dns.com`

A 레코드와 CNAME을 모두 설정하면 `myname.com`과 `www.myname.com` 둘 다 작동합니다. Vercel이 자동으로 하나를 다른 쪽으로 리다이렉트해줍니다.

DNS 변경이 전 세계에 반영되기까지 최대 48시간이 걸릴 수 있습니다. 대부분은 10분–1시간 이내에 완료됩니다. Vercel 대시보드의 Domains 페이지에서 상태를 확인할 수 있습니다. **Valid Configuration**이라고 표시되면 성공입니다.

## 흔한 문제와 해결법

비개발자가 Vercel 배포 과정에서 자주 겪는 문제와 해결 방법을 정리했습니다.

### "404: NOT_FOUND" 에러가 뜹니다

가장 흔한 원인은 `index.html`이 저장소 루트에 없는 경우입니다.

**확인 방법**: GitHub 저장소 메인 페이지에서 `index.html`이 바로 보이는지 확인합니다. `src/index.html`이나 `public/index.html`처럼 하위 폴더에 있다면, Vercel 프로젝트 설정에서 **Root Directory**를 해당 폴더(예: `src` 또는 `public`)로 지정합니다.

### CSS나 이미지가 깨져 보입니다

파일 경로 문제입니다. HTML에서 CSS나 이미지를 참조할 때 절대 경로를 사용했는지 확인합니다.

```html
<!-- 문제가 되는 경로 -->
<link rel="stylesheet" href="C:\Users\내이름\Desktop\style.css">

<!-- 올바른 상대 경로 -->
<link rel="stylesheet" href="./style.css">

<!-- 또는 절대 경로 -->
<link rel="stylesheet" href="/style.css">
```

로컬 컴퓨터의 절대 경로(`C:\Users\...` 또는 `/Users/...`)를 사용하면 배포 환경에서는 작동하지 않습니다. 반드시 상대 경로(`./`)나 루트 기준 경로(`/`)를 사용해야 합니다.

### GitHub 저장소가 Vercel에서 보이지 않습니다

Vercel의 GitHub 앱 권한 문제입니다.

1. GitHub에서 **Settings** → **Applications** → **Vercel**을 찾습니다.
2. **Configure** 버튼을 클릭합니다.
3. **Repository access** 섹션에서 해당 저장소를 추가하거나, **All repositories**를 선택합니다.

### 배포는 성공인데 빈 화면이 나옵니다

`index.html` 파일명의 대소문자를 확인합니다. `Index.html`이나 `INDEX.HTML`은 인식되지 않습니다. 반드시 소문자 `index.html`이어야 합니다.

## Vercel vs 다른 플랫폼 비교

정적 사이트를 배포할 수 있는 무료 플랫폼은 여러 개 있습니다. 비개발자 관점에서 주요 차이점을 비교합니다.

| 플랫폼 | 장점 | 단점 | 비개발자 적합도 |
|--------|------|------|----------------|
| **Vercel** | 자동 감지, 가장 쉬운 설정 | 무료 플랜 상업적 사용 제한 | ★★★★★ |
| **Netlify** | Vercel과 유사, 폼 기능 내장 | 설정 항목이 약간 더 많음 | ★★★★☆ |
| **GitHub Pages** | 완전 무료, GitHub 통합 | 설정이 다소 복잡, 빌드 속도 느림 | ★★★☆☆ |
| **Cloudflare Pages** | 무제한 대역폭, 빠른 속도 | 초기 설정이 기술적 | ★★★☆☆ |

개인 프로젝트나 포트폴리오용이라면 Vercel이 가장 빠르게 시작할 수 있습니다. 상업적 프로젝트라면 Vercel Pro 플랜(월 $20)을 고려하거나, 무료 상업 사용이 가능한 Cloudflare Pages를 대안으로 검토합니다.

## 정적 사이트 생성기(SSG)를 쓰는 경우

HTML/CSS를 직접 작성하는 대신, Hugo, Jekyll, Astro 같은 정적 사이트 생성기를 사용하는 경우도 있습니다. [AI 기반 SEO 콘텐츠 전략](/posts/ai-seo-content-strategy/)에서 다룬 것처럼, 콘텐츠 중심 사이트에서는 SSG가 효율적인 선택입니다.

좋은 소식은, Vercel이 대부분의 SSG를 자동으로 인식한다는 것입니다.

| SSG | Vercel 자동 감지 | 빌드 명령어 | 출력 폴더 |
|-----|-----------------|------------|----------|
| Hugo | 지원 | `hugo` | `public` |
| Jekyll | 지원 | `jekyll build` | `_site` |
| Astro | 지원 | `astro build` | `dist` |
| 11ty (Eleventy) | 지원 | `eleventy` | `_site` |
| Gatsby | 지원 | `gatsby build` | `public` |

SSG를 사용한다면 2단계(Import)에서 Vercel이 프레임워크를 자동 감지하고 설정을 채워줍니다. 그대로 두고 Deploy를 누르면 됩니다.

다만 SSG는 Node.js 또는 Ruby 같은 런타임이 필요합니다. Vercel 빌드 환경에 이미 설치되어 있으므로 로컬에서 설치할 필요는 없지만, `package.json`(Node.js 기반 SSG)이나 `Gemfile`(Jekyll) 같은 의존성 파일이 저장소에 포함되어 있어야 합니다.

## 한 걸음 더: 알아두면 유용한 Vercel 기능

### Preview 배포

GitHub에서 브랜치(branch)를 만들어 작업하면, Vercel이 해당 브랜치에 대한 별도 URL을 자동으로 생성합니다. 이것을 **Preview 배포**라고 합니다. 본 사이트를 건드리지 않고 변경 사항을 미리 확인할 수 있습니다.

비개발자에게 이 기능이 유용한 이유는, 실수로 사이트를 망가뜨릴 걱정 없이 수정을 시도할 수 있다는 것입니다. Preview URL로 확인한 뒤 문제가 없으면 main 브랜치에 합치면(Merge) 실제 사이트에 반영됩니다.

### 환경 변수

Google Analytics 추적 코드나 API 키처럼 코드에 직접 넣고 싶지 않은 값이 있다면, Vercel의 환경 변수(Environment Variables) 기능을 사용합니다. Settings → Environment Variables에서 설정할 수 있습니다.

### 웹 애널리틱스

Vercel은 자체 웹 애널리틱스 기능을 제공합니다. Google Analytics를 설정하기 어려운 비개발자에게 유용합니다. 프로젝트 대시보드의 **Analytics** 탭에서 방문자 수, 페이지뷰, 지역별 트래픽을 확인할 수 있습니다. 다만 무료 플랜에서는 제한된 데이터만 제공됩니다.

## AI 도구로 정적 사이트를 만드는 시대

2026년 현재, [Claude Code를 비개발자도 활용할 수 있게 된 것](/posts/claude-code-harness-non-developers-marketers-pm-2026/)처럼, 웹사이트 제작 자체도 AI 도구로 빠르게 할 수 있습니다. ChatGPT, Claude, Cursor 같은 도구에 "포트폴리오 웹사이트를 만들어줘"라고 요청하면 HTML, CSS, JavaScript 코드를 생성해줍니다.

[AI 코드 리뷰 자동화](/posts/ai-code-review-automation/)에서 다룬 것처럼, 코드의 품질을 AI가 검증하는 것도 가능합니다. 비개발자라도 AI로 사이트를 만들고, Vercel로 배포하고, AI로 코드를 검증하는 전체 워크플로우가 현실이 되었습니다.

과정을 정리하면 이렇습니다.

1. **AI에게 사이트 코드를 요청합니다.** "포트폴리오 사이트를 만들어줘. HTML과 CSS만 사용해줘."
2. **생성된 파일을 GitHub에 올립니다.** 위에서 설명한 방법 그대로 진행합니다.
3. **Vercel에서 배포합니다.** Import → Deploy. 2분이면 끝납니다.
4. **수정이 필요하면 AI에게 다시 요청합니다.** "헤더 색상을 파란색으로 바꿔줘."
5. **수정된 파일을 GitHub에 올립니다.** Vercel이 자동으로 새 버전을 배포합니다.

코딩을 모르는 것이 더 이상 웹사이트를 만들지 못하는 이유가 되지 않습니다. 도구가 바뀌었고, 진입 장벽이 사실상 사라졌습니다.

## 핵심 요약

지금 바로 실행할 수 있는 단계별 체크리스트입니다.

- **GitHub 가입**: [github.com](https://github.com)에서 무료 계정 생성
- **저장소 생성**: New repository → 파일 업로드 (`index.html`이 루트에 위치)
- **Vercel 가입**: [vercel.com](https://vercel.com)에서 GitHub 계정으로 가입
- **프로젝트 Import**: Import Git Repository → 저장소 선택 → 설정 건드리지 않고 Deploy
- **사이트 확인**: `프로젝트명.vercel.app`으로 접속
- **(선택) 커스텀 도메인**: Settings → Domains → DNS 설정

서버를 구매하거나, FTP를 설정하거나, 호스팅 업체에 전화할 필요 없습니다. GitHub에 파일을 올리고, Vercel에서 Import 하면 끝입니다. 수정할 때도 GitHub에서 파일만 바꾸면 자동으로 반영됩니다.

웹사이트를 만들었다면, 이제 세상에 보여줄 차례입니다. 배포는 더 이상 개발자만의 영역이 아닙니다.
