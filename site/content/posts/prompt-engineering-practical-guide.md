## 프롬프트 엔지니어링이란

프롬프트 엔지니어링(Prompt Engineering)은 LLM(Large Language Model, 대규모 언어 모델)에게 원하는 결과를 얻기 위해 입력을 체계적으로 설계하는 기술입니다.

단순히 질문을 잘 하는 것을 넘어, 모델의 동작 방식을 이해하고 이를 활용하는 엔지니어링 분야로 자리잡았습니다.

## 기본 원칙

### 1. 명확성 (Clarity)

모호한 지시는 모호한 결과를 만듭니다.

```
나쁜 예: "이 코드를 개선해줘"
좋은 예: "이 Python 함수의 시간 복잡도를 O(n^2)에서 O(n log n)으로 개선해줘.
         기존 인터페이스는 유지하고, 변경 사항을 주석으로 설명해줘."
```

### 2. 구조화 (Structure)

입력의 구조가 곧 출력의 구조를 결정합니다.

- 역할 정의: "당신은 10년 경력의 백엔드 엔지니어입니다"
- 맥락 제공: 관련 코드, 문서, 제약 조건
- 작업 지시: 구체적인 행동과 기대 결과
- 출력 형식: JSON, 마크다운, 코드 등 원하는 형식 명시

### 3. 제약 조건 (Constraints)

모델이 하지 말아야 할 것을 명시하는 것도 중요합니다.

- "외부 라이브러리를 추가하지 마세요"
- "기존 API 인터페이스를 변경하지 마세요"
- "추측하지 말고, 확실하지 않으면 모른다고 답하세요"

## 고급 기법

### Chain-of-Thought (CoT)

모델에게 단계별로 사고하도록 유도하는 기법입니다.

```
"이 문제를 단계별로 분석해주세요:
1. 먼저 현재 코드의 문제점을 파악하세요
2. 가능한 해결 방안을 3가지 제시하세요
3. 각 방안의 장단점을 비교하세요
4. 최적의 방안을 선택하고 구현하세요"
```

수학적 추론, 논리적 분석, 복잡한 의사결정에서 정확도가 크게 향상됩니다.

### Few-Shot Prompting

원하는 출력의 예시를 제공하여 패턴을 학습시키는 기법입니다.

```
"다음 형식으로 코드 리뷰를 작성해주세요:

[예시 입력] function add(a, b) { return a + b }
[예시 출력]
- 심각도: Low
- 카테고리: 타입 안전성
- 설명: 매개변수 타입이 명시되지 않아 런타임 에러 가능성
- 제안: TypeScript 타입 어노테이션 추가

이제 다음 코드를 리뷰해주세요: ..."
```

### System Prompt 설계

System Prompt는 모델의 전체 행동 방식을 정의합니다.

효과적인 System Prompt의 구조:

1. **역할과 전문성**: 모델이 어떤 전문가인지 정의
2. **행동 규칙**: 응답 시 따라야 할 규칙
3. **출력 형식**: 일관된 응답 형식 지정
4. **금지 사항**: 하지 말아야 할 행동 명시

### 프롬프트 체이닝

복잡한 작업을 여러 단계의 프롬프트로 분해하여 순차적으로 실행하는 기법입니다.

| 단계 | 프롬프트 | 목적 |
|------|---------|------|
| 1단계 | 요구사항 분석 | 입력을 구조화된 명세로 변환 |
| 2단계 | 설계 | 명세를 기반으로 아키텍처 설계 |
| 3단계 | 구현 | 설계를 코드로 변환 |
| 4단계 | 검증 | 구현을 명세와 대조하여 검증 |

## 실무 적용 팁

### 반복적 개선

프롬프트는 한 번에 완성되지 않습니다. 반복적으로 개선하는 과정이 필요합니다:

1. 초기 프롬프트 작성
2. 결과 평가 (기대와 실제의 차이 분석)
3. 프롬프트 수정 (부족한 지시 추가, 모호한 표현 제거)
4. 재실행 및 비교

### 프롬프트 버전 관리

프로덕션 환경에서 사용하는 프롬프트는 코드와 동일하게 버전 관리해야 합니다.

- Git으로 프롬프트 변경 이력 추적
- A/B 테스트로 프롬프트 성능 비교
- 프롬프트 변경 시 회귀 테스트 실행

### 비용 최적화

프롬프트 길이는 직접적으로 API 비용에 영향을 미칩니다.

- 불필요한 예시 제거
- 핵심 지시만 포함
- 캐싱 가능한 System Prompt 활용
- 모델별 최적 프롬프트 길이 파악

## 도구별 특성

### Claude

- XML 태그를 활용한 구조화에 강점
- 긴 컨텍스트 처리 능력이 우수
- 지시 사항 준수도가 높음

### GPT

- JSON 모드 지원으로 구조화된 출력에 유리
- Function Calling과의 통합이 용이
- 커뮤니티 리소스가 풍부

## 시작하기

프롬프트 엔지니어링을 체계적으로 학습하려면:

1. **공식 문서 숙지**: 각 모델 제공사의 프롬프트 가이드 정독
2. **실험 환경 구축**: Playground나 API로 다양한 프롬프트 실험
3. **패턴 수집**: 효과적이었던 프롬프트 패턴을 라이브러리로 관리
4. **측정 체계 수립**: 프롬프트 품질을 정량적으로 평가하는 기준 마련
